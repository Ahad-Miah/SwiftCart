
## ১) null এবং undefined-এর মধ্যে পার্থক্য

| বৈশিষ্ট্য | `undefined` | `null` |
|----------|-------------|--------|
| অর্থ | ভেরিয়েবল ঘোষণা করা হলেও কোনো মান দেওয়া হয়নি | প্রোগ্রামার ইচ্ছাকৃতভাবে "মান নেই" বোঝাতে ব্যবহার করে |
| টাইপ | `undefined` | `object` |
| ব্যবহার | জাভাস্ক্রিপ্ট স্বয়ংক্রিয়ভাবে নির্ধারণ করে | স্পষ্টভাবে মান না থাকার জন্য বরাদ্দ করা হয় |

**উদাহরণ:**
```javascript
let a;
console.log(a); // undefined

let b = null;
console.log(b); // null

## ২) map() ফাংশনের ব্যবহার এবং forEach()-এর পার্থক্য

### map() ফাংশন
- প্রতিটি অ্যারের উপাদানের ওপর কাজ করে।
- একটি **নতুন অ্যারে** রিটার্ন করে।
- মূল অ্যারে পরিবর্তন করে না।  

### forEach() ফাংশন
- প্রতিটি অ্যারের উপাদানের ওপর কাজ করে।
- কোনো মান রিটার্ন করে না (`undefined` রিটার্ন করে)।
- সাধারণত **সাইড এফেক্টের জন্য** ব্যবহার হয় (যেমন লগিং বা ডেটাবেস আপডেট)।  

### মূল পার্থক্য

| বৈশিষ্ট্য | map() | forEach() |
|-----------|-------|-----------|
| রিটার্ন মান | নতুন অ্যারে | undefined |
| মূল অ্যারে পরিবর্তন | না | না (কিন্তু চাইলে পরিবর্তন করা যায়) |
| প্রয়োগের উদ্দেশ্য | নতুন ডেটা তৈরি | শুধুমাত্র কাজ সম্পাদন করা |

### উদাহরণ
```javascript
const numbers = [1, 2, 3];

// map() ব্যবহার
const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6]

// forEach() ব্যবহার
numbers.forEach(n => console.log(n));
// Output:
// 1
// 2
// 3
